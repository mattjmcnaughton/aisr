# Plan New Feature

Plan the implementation of a new feature using the appropriate subagents.

## Arguments
- $ARGUMENTS: Description of the feature to implement

## Process
1. **Understand**: Analyze the feature requirements from $ARGUMENTS
2. **Research**: Explore the existing codebase to understand current patterns
3. **Design**: Create a detailed implementation plan
4. **Breakdown**: Split into backend, frontend, and testing tasks

## Deliverables
Create a plan that includes:
- [ ] Database schema changes (if any)
- [ ] Backend API endpoints needed
- [ ] Frontend components and state management
- [ ] E2E test scenarios
- [ ] Estimated complexity (S/M/L)

Use the Task tool with appropriate subagents:
- `db-architect` for schema design
- `backend-dev` for API planning
- `frontend-dev` for UI planning
- `e2e-tester` for test planning

Present the plan and ask for approval before implementation.
